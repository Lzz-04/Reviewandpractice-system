# 系统模块说明（开发者文档）

## 项目信息

| 项目 | 说明 |
|------|------|
| 名称 | 期末刷题助手（Exam Review System） |
| 架构 | 前后端分离：Spring Boot 3.2.5 + Nuxt 3（SPA） |
| 数据库 | MySQL 8.x，9 张表，MyBatis-Plus ORM |
| 认证 | Spring Security + JWT（jjwt 0.12.5）+ BCrypt |
| 包结构 | `com.examreview`，标准 Controller → Service → Mapper 三层 |

---

## 一、认证授权模块

### 后端

| 文件 | 路径 |
|------|------|
| Controller | `controller/AuthController.java` |
| Service | `service/AuthService.java` `service/impl/AuthServiceImpl.java` |
| DTO | `dto/LoginRequest.java` `dto/RegisterRequest.java` `dto/LoginResponse.java` |
| Entity | `entity/User.java` |
| Mapper | `mapper/UserMapper.java` |
| 安全 | `security/UserPrincipal.java` `security/UserDetailsServiceImpl.java` |
| 过滤器 | `security/JwtAuthenticationFilter.java` |
| 工具 | `util/JwtUtil.java` `util/SecurityUtil.java` |

### 接口

| 方法 | 路径 | 说明 | 认证 |
|------|------|------|------|
| POST | `/api/auth/register` | 注册。校验用户名 2-50 字符、密码 6-100 字符。BCrypt 加密入库，返回 JWT | 否 |
| POST | `/api/auth/login` | 登录。Spring Security AuthenticationManager 认证，返回 JWT | 否 |
| GET | `/api/auth/me` | 获取当前登录用户信息（通过 JWT Token 解析 userId） | 是 |

### 前端

| 文件 | 说明 |
|------|------|
| `pages/login.vue` | 登录表单，调用 `authStore.login()` |
| `pages/register.vue` | 注册表单，调用 `authStore.register()` |
| `stores/auth.js` | Pinia Store。token + user 状态，localStorage 持久化 |
| `middleware/auth.global.js` | 全局路由守卫。未登录 → `/login`，已登录访问登录页 → `/` |
| `composables/useApi.js` | API 封装。自动从 localStorage 读取 Token 注入请求头。401 时自动清除登录态并跳转登录页 |

### 关键实现

- 密码使用 `BCryptPasswordEncoder` 加密，数据库存储 `$2a$10$...` 格式
- JWT 使用 HMAC-SHA 签名，24 小时过期，payload 包含 `sub`（userId）和 `username`
- `JwtAuthenticationFilter` 继承 `OncePerRequestFilter`，从 `Authorization: Bearer <token>` 提取 Token，校验后设置 `SecurityContext`
- `DataInitializer` 在首次启动时自动创建 `admin/123456` 管理员账户

- 批量删除事务：`for (Integer id : ids) { delete(id); }` 由 `@Transactional` 保证原子性

### 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/exams` | 试卷列表。支持 subjectId 筛选 |
| POST | `/api/exams/generate` | 自动组卷。参数 subjectId + title + duration + totalCount + chapterIds（可选） |
| POST | `/api/exams/{id}/start` | 开始考试。生成 ExamRecord，状态 `in_progress` |
| POST | `/api/exams/submit` | 交卷。`@Transactional`：逐题评分 → 保存 AnswerRecord → 错题同步 WrongBook → 更新 ExamRecord |
| GET | `/api/exams/{id}/questions` | 获取试卷题目（按 sort_order） |
| DELETE | `/api/exams/{id}` | 删除。级联删除 answer_records / exam_records / exam_questions |
| GET | `/api/exams/records` | 考试记录分页（按当前用户） |

| POST | `/api/exams/records/{id}/pause` | 暂停。参数 remainingSeconds，状态 `paused` |
| POST | `/api/exams/records/{id}/resume` | 恢复。状态 `in_progress`，重置计时起点 |

### 前端

| 文件 | 说明 |
|------|------|
| `pages/exam/index.vue` | 考试列表 + 快速组卷面板 |
| `pages/exam/start/[examId].vue` | 考试答题。倒计时 + 答题卡网格 + 暂停/交卷按钮 |
| `pages/exam/result/[recordId].vue` | 考试结果。得分环形图 + 统计 + 逐题解析 |
| `components/exam/ExamTimer.vue` | 倒计时组件。正常 / 警告（<5min 黄）/ 危急（<1min 红脉冲） |
| `composables/useExamTimer.js` | 计时逻辑。start/pause/stop，formatTime 计算属性，时间到回调 |
| `stores/examStore.js` | 考试会话状态。examId / recordId / questions / answers / currentIndex / duration |

### 关键实现

- 组卷算法：按科目查询题目池 → `Collections.shuffle()` → 按题型分组 → 循环从各题型取题（单→多→判→单...）直到达到 totalCount
- 暂停恢复：暂停时计算已消耗时间保存到 `duration_used`，设置 `duration_remaining`；恢复时重置 `started_at` 为当前时间
- 交卷 sessionId：`"exam-" + recordId + "-" + System.currentTimeMillis()`
- 评分：`correctCount / totalQuestions * 100`，`BigDecimal` 保留 1 位小数
- 交卷后错题同步：遍历 examQuestions → `answerChecker.checkAnswer()` → 错误调用 `wrongBookService.upsertWrongQuestion()`

## 数据库表关系

```
users
  ├── subjects ──→ chapters ──→ questions
  │                                │
  ├── exam_records ←── exam_papers ──→ exam_questions ←── questions
  │       │
  ├── answer_records ←── questions
  │
  └── wrong_questions ←── questions
```

### 表清单

| 表 | 主键 | 核心外键/唯一约束 |
|------|------|------|
| `users` | id (BIGINT) | username UNIQUE |
| `subjects` | id (INT) | user_id, name UNIQUE |
| `chapters` | id (INT) | subject_id FK, user_id |
| `questions` | id (INT) | chapter_id FK, subject_id FK, user_id |
| `exam_papers` | id (INT) | subject_id FK |
| `exam_questions` | id (INT) | exam_id FK, question_id FK |
| `exam_records` | id (INT) | exam_id FK, user_id, session_id |
| `answer_records` | id (INT) | question_id FK, exam_id, session_id, user_id |
| `wrong_questions` | id (INT) | user_id + question_id UNIQUE |
