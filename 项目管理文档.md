# 期末刷题助手 — 项目管理文档

## 项目简介

"期末刷题助手"是一个面向大学生的 Web 刷题平台。系统采用前后端分离架构（Spring Boot 3 + Nuxt 3），支持多用户在线使用。学生可以按科目和章节管理题库，通过顺序、随机、错题三种模式进行日常练习，也可以参与带有倒计时和暂停恢复功能的模拟考试。答题过程中产生的错题会自动收录到错题本，方便针对性复习。系统还集成了 DeepSeek 大模型，支持 AI 智能出题。管理员可通过后台查看所有用户数据并进行管理。

**核心特点：**

- 📚 **完整题库管理** — 科目 → 章节 → 题目三层结构，支持单选/多选/判断三种题型，JSON 格式存储选项，批量操作含事务回滚
- 📥 **多格式文件导入** — 支持 XLSX、DOCX、PDF、TXT 四种格式批量导入题目，两种解析模式自动识别
- ✏️ **三种练习模式** — 顺序刷题、随机抽题、错题重练，逐题即时反馈，答错自动收录错题本
- 📋 **全流程模拟考试** — 自动组卷（按题型均匀分配）→ 倒计时答题 → 暂停恢复 → 交卷自动评分 → 成绩详情
- 📕 **错题本自动管理** — 练习和考试的错题自动收集，支持复习标记和掌握状态切换，用户数据完全隔离
- 🤖 **AI 智能出题** — 调用 DeepSeek 大模型，按科目/章节/题型/数量自动生成题目
- 📊 **学习数据可视化** — ECharts 图表展示正确率趋势、每日活跃度、各科目进度
- 🔐 **安全认证** — JWT + BCrypt，登录有效期管理，路由守卫，401 自动拦截跳转

## 二、系统架构

```
┌───────────────────────────────────┐
│         浏览器 (Nuxt 3 SPA)        │
│   Element Plus + Pinia + Vue 3    │
│          localhost:3000           │
└──────────────┬────────────────────┘
               │ HTTP REST / JSON
               │ Authorization: Bearer <JWT>
┌──────────────┴────────────────────┐
│      后端 (Spring Boot 3.2.5)      │
│   Security + JWT + MyBatis-Plus   │
│           localhost:8080          │
└──────────────┬────────────────────┘
               │ JDBC
┌──────────────┴────────────────────┐
│        MySQL (exam_review)        │
│          localhost:3306           │
└───────────────────────────────────┘
```

**认证流程：** 用户登录 → 后端验证 → 返回 JWT Token → 前端存入 localStorage → 后续请求由 `useApi.js` 自动注入 `Authorization: Bearer <token>` → `JwtAuthenticationFilter` 校验 → `SecurityContext` 设置用户身份。

### 1. 认证授权

用户注册（用户名 2-50 字符 / 密码 6-100 字符 / 可选昵称），密码 BCrypt 加密入库。登录时 Spring Security AuthenticationManager 认证，通过后颁发 JWT Token（24h 有效期）。所有业务接口（除 login/register）均需携带 `Authorization: Bearer <token>`。前端路由守卫拦截未登录访问，401 响应自动清除登录态并跳转登录页。系统首次启动自动创建默认管理员 `admin / 123456`。

### 2. 科目管理

学科分类的增删改查，如"高等数学""大学英语"。创建和编辑时校验名称非空、≤50 字符、无重名（编辑时排除自身）。删除时检查关联章节数，>0 则阻止删除并提示具体数量。列表按 `sort_order` 排序展示。

### 3. 章节管理

科目下的单元划分，如"第一章 极限与连续"。创建时自动分配排序号（当前科目最大 +1）。编辑支持修改名称和排序。删除时双重检查：关联题目数 >0 或关联错题数 >0 均阻止删除并提示。

### 4. 题库管理

系统核心数据模块。支持单选（single）、多选（multiple）、判断（judge）三种题型，选项以 JSON 格式 `[{"label":"A","text":"..."}]` 存储。分页列表支持科目/章节/题型/难度/关键词多条件组合筛选，可按难度排序。创建时校验题型合法性。支持批量删除（`@Transactional` 事务回滚）、批量移动章节、批量修改难度。随机抽题用于练习和组卷。删除题目时级联清理关联的错题记录、答题记录和试卷题目关联。

### 5. 文件导入

支持 XLSX / DOCX / PDF / TXT 四种格式批量导入题目。内置两种解析模式自动识别：标准标签格式（题型：/题目：/A.选项/答案：/解析：）和试卷格式（题号 + 行内选项 + 答案区）。导入前支持预览（提取前 10 道单选题不写入数据库），导入后返回成功/失败计数和错误详情。

### 6. 章节练习

日常刷题核心功能。支持三种模式：顺序练习（按 ID）、随机练习（Collections.shuffle）、错题练习（仅未掌握题目）。逐题展示 → 选择答案 → 即时判对错 → 显示正确答案和解析。答错自动调用 `upsertWrongQuestion` 写入错题本。每题记录耗时和正确性，使用 sessionId 分批关联。

### 7. 模拟考试

完整的考试流程模拟。自动组卷：按科目筛选题库 → 随机打乱 → 按题型循环分配（single → multiple → judge）直到达到目标题数。考试生命周期：开始生成记录 → 答题卡网格导航 → 暂停（保存已消耗时间和剩余时间）→ 恢复（重置计时段点）→ 交卷。同一用户同一试卷同时只能有一个进行中记录。交卷后逐题比对答案，百分制评分（保留1位小数），sessionId 精确关联当次答题记录。可视化倒计时组件：剩余 <5 分钟黄色警告，<1 分钟红色脉冲，时间到自动交卷。结果页展示得分环形图和逐题解析。

### 8. 错题本

练习和考试中的错题自动收集到错题本，同一道题多次做错累加错误次数。分页列表关联 Question 表获取题目详情，支持按科目和掌握状态筛选。操作包括：复习标记（reviewedCount++）、掌握状态切换（0↔1）、移除错题。统计卡片显示总数/已掌握/未掌握。通过 SecurityUtil.getCurrentUserId() 进行所有权校验，用户只能操作自己的错题。

### 9. 学习统计

学习数据可视化展示。概览卡片：总题数、已答题数、整体正确率、累计学习天数、今日答题数。ECharts 折线图展示 7 日正确率趋势，柱状图展示 30 日每日活跃度，进度条展示各科目完成情况。所有统计数据按 userId 隔离。

### 10. AI 辅助

调用 DeepSeek 大模型的公共服务模块，被考试、统计、错题本等模块注入使用。AI 出题：指定科目/章节/题型/数量 → 结构化 Prompt → 返回 JSON 格式题目 → 前端预览勾选确认后逐条入库。AI 错题解析：对错题提供详细解答。AI 考试总结：交卷后综合分析。AI 学习规划：基于历史数据的个性化复习建议。

### 11. 管理后台

管理员专属模块（role='admin'）。用户列表展示所有注册用户及数据统计（科目数/题目数/考试数/错题数）。支持级联删除用户：删除用户时同步清理其所有关联数据（subjects → chapters → questions → wrong_questions → exam_records → answer_records）。

## 四、数据库设计

### E-R 关系

```
users 1──N subjects 1──N chapters 1──N questions
                     │
users 1──N exam_records ──N:1── exam_papers 1──N exam_questions N──1 questions
                     │
users 1──N wrong_questions N──1 questions
                     │
users 1──N answer_records N──1 questions
```

### 表清单

| 表名 | 说明 | 核心字段 |
|------|------|----------|
| `users` | 用户 | id, username (UNIQUE), password (BCrypt), nickname, role |
| `subjects` | 科目 | id, name, description, sort_order, user_id |
| `chapters` | 章节 | id, subject_id (FK), name, sort_order, user_id |
| `questions` | 题目 | id, chapter_id (FK), subject_id (FK), type (single/multiple/judge), content, options (JSON), answer, analysis, difficulty (1-5), user_id |
| `exam_papers` | 试卷 | id, subject_id, title, duration, question_count |
| `exam_questions` | 试卷-题目 | id, exam_id (FK), question_id (FK), sort_order |
| `exam_records` | 考试记录 | id, exam_id, user_id, score, correct_count, wrong_count, status (in_progress/paused/finished), duration_used, session_id |
| `answer_records` | 答题记录 | id, question_id, exam_id, session_id, user_id, selected_answer, is_correct, time_spent |
| `wrong_questions` | 错题本 | id, user_id + question_id (UNIQUE), subject_id, chapter_id, wrong_count, last_wrong_at, reviewed_count, mastered |

### 用户隔离

所有业务表均包含 `user_id` 列，查询时通过 `SecurityUtil.getCurrentUserId()` 获取当前用户 ID 进行过滤。

### 索引

- `questions`: chapter_id, subject_id, type, difficulty
- `answer_records`: session_id, question_id, exam_id, answered_at
- `wrong_questions`: user_id + question_id (UNIQUE), subject_id
- `exam_records`: exam_id, subject_id, user_id

## 六、安装部署

### 环境要求

| 组件 | 版本 |
|------|------|
| JDK | 17+ |
| Maven | 3.9+ |
| MySQL | 8.0+ |
| Node.js | 18+ |

### 步骤

**1. 数据库**

MySQL 会自动建库（`application.yml` 中 `createDatabaseIfNotExist=true`）。只需确保 MySQL 服务运行，用户名密码正确。

**2. 配置文件**

```bash
# 复制配置模板
cp server/src/main/resources/application-template.yml server/src/main/resources/application.yml
```

或设置环境变量：

| 变量 | 说明 | 必填 |
|------|------|------|
| `DB_PASSWORD` | MySQL 密码 | 是 |
| `JWT_SECRET` | JWT 签名密钥（至少 32 字符） | 是 |
| `DEEPSEEK_API_KEY` | DeepSeek API Key | 否（AI 功能需要） |

**3. 启动后端**

```bash
cd server
mvn spring-boot:run
# → http://localhost:8080
```

**4. 启动前端**

```bash
cd client
npm install
npm run dev
# → http://localhost:3000
```

**5. 登录**

默认管理员：`admin / 123456`

## 八、关键实现细节

### 答案比对逻辑

| 题型 | 比对方式 |
|------|----------|
| 单选题（single） | 字母直接比较 |
| 多选题（multiple） | 逗号分隔 → 排序 → 数组逐项比较 |
| 判断题（judge） | 大小写不敏感字符串比较（T/F） |

### 组卷算法

1. 按科目 + 可选章节筛选题目池
2. `Collections.shuffle()` 随机打乱
3. 按题型循环分配：single → multiple → judge → single → ... 确保各题型均匀分布
4. 题目不足时抛出 BusinessException 提示

### 暂停/恢复考试

- **暂停**：保存 `duration_remaining`，状态设为 `paused`，记录已消耗时间
- **恢复**：状态还原为 `in_progress`，重置计时段点
- **答题进度**：通过 `answer_records` 表持久化，恢复时重新加载
- **sessionId**：交卷时生成，用于精确关联当次考试的答题记录

### 文件导入解析

| 格式 | 解析方式 |
|------|----------|
| XLSX | Apache POI 读取单元格 |
| DOCX | Apache POI 提取段落文本 |
| PDF | Apache PDFBox 提取文本 |
| TXT | 直接读取文本 |

支持两种文本解析模式：标准标签格式和试卷格式，自动识别并转换。

### 用户隔离

业务查询通过 `SecurityUtil.getCurrentUserId()` 获取当前用户，Service 层使用 `LambdaQueryWrapper.eq(Entity::getUserId, userId)` 过滤。管理员通过 `role='admin'` 跳过过滤。

### 事务管理

- 题目删除：`@Transactional` 保证级联删除关联表原子性
- 批量操作：`@Transactional` 保证任一失败则全部回滚
- 交卷：`@Transactional` 保证评分、答题记录、错题同步原子性
