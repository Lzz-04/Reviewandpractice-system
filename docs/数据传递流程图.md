## 一、登录数据传递流程

### 1.1 登录请求流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Vue as 前端(Vue)
    participant Store as Pinia Store
    participant Api as useApi
    participant Server as 后端服务器
    participant Filter as JWT过滤器
    participant Security as Spring Security
    participant Service as AuthService
    participant DB as 数据库

    User->>Vue: 输入用户名密码
    Vue->>Vue: 表单校验
    Vue->>Store: authStore.login(username, password)
    Store->>Api: api.post('/auth/login', {username, password})
    
    Api->>Api: 添加请求头<br/>Content-Type: application/json
    Api->>Server: POST http://localhost:8080/api/auth/login
    
    Server->>Filter: 拦截请求
    Filter->>Filter: 检查是否为登录接口
    Filter->>Server: 放行（登录接口无需Token）
    
    Server->>Security: AuthenticationManager.authenticate()
    Security->>Security: UserDetailsService.loadUserByUsername()
    Security->>DB: SELECT * FROM users WHERE username = ?
    DB-->>Security: 返回用户信息
    Security->>Security: BCryptPasswordEncoder.matches()
    
    alt 密码正确
        Security->>Service: 认证成功
        Service->>Service: JwtUtil.generateToken(userId, username)
        Service->>DB: 查询用户完整信息
        DB-->>Service: 返回用户详情
        Service-->>Server: 返回LoginResponse
        Server-->>Api: {code: 200, data: {token, userId, username, nickname, role}}
        Api->>Api: 检查业务状态码
        Api-->>Store: 返回data
        Store->>Store: 更新token和user状态
        Store->>Store: localStorage.setItem('auth_token', token)
        Store->>Store: localStorage.setItem('auth_user', JSON.stringify(user))
        Store-->>Vue: 返回true
        Vue->>User: 显示"登录成功"提示
        Vue->>Vue: 跳转到首页
    else 密码错误
        Security-->>Server: 抛出BadCredentialsException
        Server-->>Api: {code: 401, message: "用户名或密码错误"}
        Api-->>Store: 抛出错误
        Store-->>Vue: 返回false
        Vue->>User: 显示错误提示
    end
```


**后端返回：**

```json
    "userId": 1,

```

```mermaid
graph TD
```

## 二、科目管理数据传递流程

### 2.1 创建科目流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Vue as 前端(Vue)
    participant Api as useApi
    participant Server as 后端服务器
    participant Filter as JWT过滤器
    participant Controller as SubjectController
    participant Service as SubjectService
    participant DB as 数据库

    User->>Vue: 点击"添加科目"
    Vue->>Vue: 显示对话框
    User->>Vue: 输入科目信息<br/>name: "高等数学"<br/>description: "大学数学基础课程"<br/>icon: "📐"
    Vue->>Vue: 表单校验
    Vue->>Api: api.post('/subjects', {name, description, icon})
    
    Api->>Api: 从localStorage读取token
    Api->>Api: 添加请求头<br/>Authorization: Bearer token
    Api->>Api: 添加Content-Type: application/json
    Api->>Server: POST http://localhost:8080/api/subjects
    
    Server->>Filter: 拦截请求
    Filter->>Filter: 提取token
    Filter->>Filter: 解析token获取userId
    Filter->>Filter: 设置SecurityContext
    Filter->>Server: 放行
    
    Server->>Controller: SubjectController.create()
    Controller->>Controller: @AuthenticationPrincipal获取userId
    Controller->>Service: subjectService.create(subject, userId)
    
    Service->>Service: 校验科目名称非空
    Service->>Service: 校验科目名称长度
    Service->>DB: SELECT COUNT(*) FROM subjects WHERE name = ? AND user_id = ?
    DB-->>Service: 返回count
    Service->>Service: 检查count > 0
    
    alt 名称已存在
        Service-->>Controller: 抛出BusinessException("科目名称已存在")
        Controller-->>Server: 返回错误响应
        Server-->>Api: {code: 400, message: "科目名称已存在"}
        Api-->>Vue: 显示错误提示
    else 名称可用
        Service->>Service: subject.setUserId(userId)
        Service->>DB: INSERT INTO subjects (name, description, icon, sort_order, user_id) VALUES (?, ?, ?, 0, ?)
        DB-->>Service: 返回自增ID
        Service-->>Controller: 返回Subject对象
        Controller-->>Server: {code: 200, data: {id, name, description, icon, sortOrder, userId, createdAt}}
        Server-->>Api: 返回成功响应
        Api-->>Vue: 返回data
        Vue->>Vue: 关闭对话框
        Vue->>Vue: 重新加载科目列表
        Vue->>User: 显示"创建成功"提示
    end
```

### 2.2 获取科目列表流程

```mermaid
sequenceDiagram
    participant Vue as 前端(Vue)
    participant Api as useApi
    participant Server as 后端服务器
    participant Filter as JWT过滤器
    participant Controller as SubjectController
    participant Service as SubjectService
    participant DB as 数据库

    Vue->>Api: api.get('/subjects')
    
    Api->>Api: 从localStorage读取token
    Api->>Api: 添加请求头<br/>Authorization: Bearer token
    Api->>Server: GET http://localhost:8080/api/subjects
    
    Server->>Filter: 拦截请求
    Filter->>Filter: 解析token
    Filter->>Filter: 设置SecurityContext
    Filter->>Server: 放行
    
    Server->>Controller: SubjectController.getAll()
    Controller->>Controller: @AuthenticationPrincipal获取userId
    Controller->>Service: subjectService.getAll(userId)
    
    Service->>Service: 判断是否为管理员
    alt 普通用户
        Service->>DB: SELECT * FROM subjects WHERE user_id = ? ORDER BY sort_order ASC
    else 管理员
        Service->>DB: SELECT * FROM subjects ORDER BY sort_order ASC
    end
    DB-->>Service: 返回科目列表
    Service-->>Controller: 返回List<Subject>
    Controller-->>Server: {code: 200, data: [{id, name, description, icon, sortOrder, userId, createdAt}, ...]}
    Server-->>Api: 返回成功响应
    Api-->>Vue: 返回科目列表
    Vue->>Vue: 更新subjects响应式变量
    Vue->>Vue: 渲染科目卡片
```

### 2.3 删除科目流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Vue as 前端(Vue)
    participant Api as useApi
    participant Server as 后端服务器
    participant Controller as SubjectController
    participant Service as SubjectService
    participant DB as 数据库

    User->>Vue: 点击删除按钮
    Vue->>Vue: 显示确认对话框
    User->>Vue: 确认删除
    Vue->>Api: api.delete(`/subjects/${id}`)
    
    Api->>Api: 添加Authorization头
    Api->>Server: DELETE http://localhost:8080/api/subjects/1
    
    Server->>Controller: SubjectController.delete(id)
    Controller->>Controller: 获取userId
    Controller->>Service: subjectService.delete(id, userId)
    
    Service->>Service: getById(id, userId)
    Service->>DB: SELECT * FROM subjects WHERE id = ?
    DB-->>Service: 返回科目信息
    Service->>Service: 检查userId是否匹配
    
    Service->>DB: SELECT COUNT(*) FROM chapters WHERE subject_id = ?
    DB-->>Service: 返回章节数量
    
    alt 有关联章节
        Service-->>Controller: 抛出BusinessException("该科目下还有X个章节，请先删除章节")
        Controller-->>Server: 返回错误响应
        Server-->>Api: {code: 400, message: "该科目下还有X个章节，请先删除章节"}
        Api-->>Vue: 显示错误提示
    else 无关联章节
        Service->>DB: DELETE FROM subjects WHERE id = ?
        DB-->>Service: 删除成功
        Service-->>Controller: 返回void
        Controller-->>Server: {code: 200, message: "删除成功"}
        Server-->>Api: 返回成功响应
        Api-->>Vue: 返回成功
        Vue->>Vue: 重新加载科目列表
        Vue->>User: 显示"删除成功"提示
    end
```

## 三、章节管理数据传递流程

### 3.1 创建章节流程

```mermaid
sequenceDiagram
    participant User as 用户
    participant Vue as 前端(Vue)
    participant Api as useApi
    participant Server as 后端服务器
    participant Controller as ChapterController
    participant Service as ChapterService
    participant DB as 数据库

    User->>Vue: 点击"添加章节"

    User->>Vue: 输入章节名称
    Vue->>Api: api.post('/chapters', {name, subjectId})
    
    Api->>Api: 添加Authorization头
    Api->>Server: POST http://localhost:8080/api/chapters
    
    Server->>Controller: ChapterController.create()

    Service->>Service: 校验章节名称非空

    Service->>DB: INSERT INTO chapters (subject_id, name, sort_order, user_id) VALUES (?, ?, ?, ?)
    DB-->>Service: 返回自增ID
    Service-->>Controller: 返回Chapter对象
    Controller-->>Server: {code: 200, data: {id, subjectId, name, sortOrder, userId, createdAt}}
    Server-->>Api: 返回成功响应
    Api-->>Vue: 返回data
    Vue->>Vue: 关闭对话框
    Vue->>Vue: 重新加载章节列表
    Vue->>User: 显示"创建成功"提示
```

```mermaid
graph TD
```

## 七、数据传递关键点

### 7.1 前端关键点

1. **Token管理**
   - 登录成功后存储到localStorage
   - 每次请求自动添加到Authorization头
   - 401错误时清除Token并跳转登录页

2. **数据格式**
   - 使用JSON格式传输
   - 日期格式：ISO 8601（YYYY-MM-DDTHH:mm:ss）
   - 布尔值：true/false

3. **错误处理**
   - 统一在useApi中处理
   - 显示错误提示
   - 401自动跳转登录页

### 7.2 后端关键点

1. **认证鉴权**
   - JWT过滤器拦截所有请求
   - 解析Token设置SecurityContext
   - @AuthenticationPrincipal获取当前用户

2. **数据校验**
   - @Valid触发参数校验
   - 业务逻辑校验（如名称唯一性）
   - 抛出BusinessException返回错误信息

3. **数据隔离**
   - 普通用户只能访问自己的数据
   - 管理员可访问所有数据
   - 通过user_id字段实现

### 7.3 数据库关键点

1. **表设计**
   - 所有业务表添加user_id字段
   - 使用外键约束保证数据完整性
   - 添加created_at时间戳

2. **查询优化**
   - 使用索引加速查询
   - 分页查询避免大数据量
   - 条件查询减少数据传输

3. **事务管理**
   - @Transactional保证原子性
   - 级联删除使用事务
   - 失败时自动回滚

## 八、总结

本文档详细展示了A负责模块中数据传递的完整流程，包括：

1. **登录流程**：从用户输入到Token生成的完整过程
2. **科目管理**：创建、查询、删除科目的数据传递
3. **章节管理**：创建章节的自动排序逻辑
4. **认证流程**：JWT Token的结构和解析过程
5. **数据隔离**：通过user_id实现多用户数据隔离
6. **级联删除**：删除用户时的数据清理流程

通过这些流程图，可以清晰地理解数据在前端、后端、数据库之间的传递过程，有助于系统开发和维护。
