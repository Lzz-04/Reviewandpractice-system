# 期末刷题助手 — 四人分工方案

## 分工总览

| 成员 | 模块 | 后端 | 前端 |
|------|------|------|------|
| **A** | 认证 + 科目 + 章节 + 管理后台 | 4 个 Controller | 4 个页面 |
| **B** | 题库管理 + 文件导入 | 2 个 Controller | 1 个页面 + 弹窗 |
| **C** | 章节练习 + 错题本 + AI 辅助 | 3 个 Controller | 2 个页面 + 弹窗 |
| 李昀朔 | 模拟考试 + 学习统计 | 2 个 Controller | 4 个页面 |

## 👤 C — 刷题闭环（章节练习 / 错题本 / AI 辅助）

### 负责模块

| 模块 | 后端 | 前端 |
|------|------|------|
| 章节练习 | `PracticeController` | `practice/[chapterId].vue` |

### 章节练习功能清单

- 三种模式：顺序 / 随机 / 仅错题

- 调用 DeepSeek API 生成题目

### 依赖

- 练习依赖 B 的题库 CRUD 和随机抽题
- 错题本依赖练习和考试模块输出

## 👤 D — 考试+统计（模拟考试 / 学习统计）

### 负责模块

| 模块 | 后端 | 前端 |
|------|------|------|

### 模拟考试功能清单

- 自动组卷（按科目/章节筛选 → 随机打乱 → 题型循环分配）

- 错题自动同步到错题本

- 各科目进度对比
- 数据按 userId 隔离

### 公共组件（D 负责）

- 统计依赖练习和考试的答题记录

## ⚠️ 关键依赖链

```
A（认证 + 框架）────── 所有人依赖
        ↓
A（科目 + 章节） ──→ B（题库） ──→ C（练习）
        ↓                ↓
        │                └──→ D（考试组卷）
        │                        ↓
        └──────── C（错题本）←── D（考试交卷）
```

**绝对不能乱序的开发节点：**

1. 第 1 周结束前：A 必须交付认证 + 科目 + 章节 + 布局
2. 第 2 周结束前：B 必须交付题库 CRUD + 随机抽题
3. 第 4 周：C 和 D 必须联调考试交卷 → 错题本写入流程

## 📋 GitHub Issues 清单

### 👤 A 的 Issues（5 个）

#### #1 [Auth] 用户登录/注册（后端）

实现用户认证的后端功能。包括：注册接口（用户名 2-50 字符 / 密码 6-100 字符 / 可选昵称，BCrypt 加密入库，同名检测）；登录接口（Spring Security AuthenticationManager 认证，返回 JWT Token 24h 过期）；获取当前用户信息接口（从 JWT 解析 userId）；Spring Security 安全配置（CSRF 关闭 / 无状态会话 / JWT 过滤器 / 401/403 JSON 响应）；JwtUtil 工具类（HMAC-SHA 签名 / 生成 / 解析 / 校验）；全局异常处理器（BusinessException / MethodArgumentNotValidException / BadCredentialsException）；DataInitializer 首次启动创建 admin 管理员。

#### #2 [Auth] 登录/注册（前端）

实现用户认证的前端功能。包括：登录页面（用户名 + 密码表单 + Element Plus 校验）；注册页面（用户名 + 昵称 + 密码 + 确认密码 + 前端一致性校验）；Pinia authStore（token / user / isAuthenticated 状态管理 + localStorage 持久化）；useApi 封装（自动注入 Bearer Token + 401 拦截清除登录态并跳转）；全局路由守卫 auth.global.js（未登录 → /login，已登录访问登录页 → /）；默认布局 default.vue（侧边栏导航 + 顶栏面包屑 + 用户信息展示）。

#### #3 [Subject] 科目增删改查

实现科目管理的全栈功能。后端：SubjectController + SubjectService（GET 列表按 sort_order 排序 / GET 详情 / POST 创建校验名称非空 ≤50 字符不重名 / PUT 更新同名检测排除自身 / DELETE 删除检查关联章节数 >0 阻止）。前端：subjects.vue 科目卡片网格 + 新建/编辑弹窗 + 删除确认。

#### #4 [Chapter] 章节增删改查

实现章节管理的全栈功能。后端：ChapterController + ChapterService（GET 按科目查章节列表 / POST 创建自动分配 sort_order = 当前最大 +1 / PUT 更新 / DELETE 删除双重检查关联题目数和错题数 >0 阻止）。前端：subject/[id].vue 章节面板（左侧列表 + 切换选中）。

#### #5 [Admin] 管理后台

实现管理员专属的全栈功能。后端：AdminController（GET /api/admin/users 用户列表 + 各用户统计 / DELETE /api/admin/users/{id} 级联删除用户及所有数据）。前端：admin.vue 用户表格 + 删除按钮确认弹窗。仅 admin 角色可访问。

#### #7 [Import] 文件导入

实现文件批量导入题目的全栈功能。后端：ImportController + ImportService。支持 XLSX（POI）/ DOCX（POI）/ PDF（PDFBox）/ TXT 四种格式。两种文本解析模式自动识别：标准标签格式（正则匹配题型：/题目：/A./B./C./D./答案：/解析：）和试卷格式（题号 + 内联选项 + 答案区）。POST /api/import 执行导入返回 ImportResultDTO{total, success, failed, errors[]}。POST /api/import/preview 仅提取前 10 道单选题不写入。前端：subject/[id].vue 中的导入弹窗（el-upload 选文件 → 选目标章节 → 预览 → 确认导入 → 展示结果统计）。

### 👤 C 的 Issues（3 个）

#### #8 [Practice] 章节练习

实现章节练习的全栈功能。后端：PracticeController。GET /start/{chapterId} 开始练习，返回题目列表（支持 mode=sequential/random 参数）。GET /wrong/{subjectId} 错题专项练习返回未掌握题目（上限 50）。POST /answer 提交答案：AnswerChecker 比对（单选直接比较 / 多选逗号分隔排序后数组比较 / 判断大小写不敏感）→ 返回正确/错误 + 正确答案 + 解析。答错自动调用 wrongBookService.upsertWrongQuestion 写入错题本。每题记录 timeSpent + sessionId。前端：practice/[chapterId].vue 练习页（模式选择 → 进度条 → QuestionCard 通用组件 → 上一题/下一题/确认 → 完成汇总）。QuestionCard 通用组件：单选 radio / 多选 checkbox / 判断 T/F，选中/正确/错误状态高亮。questionStore Pinia 会话状态管理。

#### #9 [WrongBook] 错题本管理

实现错题本的全栈功能。后端：WrongBookController + WrongBookService。GET 分页列表关联 Question 表查询题目详情，支持 subjectId/mastered 筛选。GET /stats 返回 {total, mastered, unMastered}。POST /{id}/review 复习标记 reviewedCount++。POST /{id}/master 切换掌握状态 0↔1。DELETE /{id} 移除，通过 SecurityUtil.getCurrentUserId() 校验所有权。upsertWrongQuestion 方法：已存在则 wrongCount++ 并更新 lastWrongAt，不存在则插入新记录。前端：wrongbook.vue（统计卡片 + 筛选栏 + 错题列表 + 复习/掌握/移除操作 + 分页）。

#### #10 [AI] DeepSeek 出题

实现 AI 智能出题的全栈功能。后端：AIController + AIService。使用 Spring WebClient 调用 DeepSeek Chat API（兼容 OpenAI 格式）。结构化 Prompt：系统提示指定 JSON 格式 + 用户提示指定科目/章节/题型/数量。配置 response_format: json_object 确保返回合法 JSON。POST /api/ai/generate 返回题目列表供预览。前端：subject/[id].vue 中的 AI 出题弹窗（配置科目/章节/题型/数量 → 生成 → 预览勾选 → 逐条调用 POST /api/questions 确认入库）。

### 👤 D 的 Issues（5 个）

#### #11 [Exam] 自动组卷

实现智能组卷功能。后端：ExamController + ExamService。POST /api/exams/generate 接收 ExamGenerateDTO（subjectId + title + duration + totalCount + 可选 chapterIds）。算法：按科目查询题目池 → 可选按章节过滤 → Collections.shuffle() 随机打乱 → 按题型分组 → 循环从各题型取题（single → multiple → judge → single...）直到达到 totalCount → 创建 ExamPaper + ExamQuestion 关联记录。题目不足时抛出 BusinessException 提示。前端：exam/index.vue 快速组卷面板（选科目/标题/时长/题数 → 一键生成）。

#### #12 [Exam] 考试答题流程

实现考试的核心流程。后端：ExamController + ExamService。POST /{id}/start 生成 ExamRecord（status=in_progress，同一用户同一试卷只能有一个进行中）。POST /records/{id}/pause 暂停：计算已消耗时间保存到 duration_used + 保存 duration_remaining + status=paused。POST /records/{id}/resume 恢复：重置 started_at + status=in_progress。POST /submit 交卷 @Transactional：遍历 examQuestions → AnswerChecker 比对 → 正确 correctCount++ / 错误 wrongCount++ 且调用 wrongBookService.upsertWrongQuestion → 保存 AnswerRecord（含 sessionId）→ 更新 ExamRecord（score=correctCount/totalQuestions*100 保留1位小数 + status=finished + sessionId="exam-"+recordId+"-"+timestamp）。GET /{id}/questions 按 sort_order 返回试卷题目。前端：exam/start/[examId].vue 答题页（倒计时 + 答题卡网格导航 + 暂停/交卷按钮 + 恢复弹窗）。examStore Pinia 考试状态管理。

#### #13 [Exam] 考试结果页

实现考试结果展示。后端：ExamController.getRecordDetail() 通过 sessionId 精确关联当次 AnswerRecord，组装 ExamResultDTO（score / correctCount / wrongCount / durationUsed + questions[] 每题 QuestionResultItem 含 questionIndex / content / options / correctAnswer / selectedAnswer / isCorrect / analysis）。前端：exam/result/[recordId].vue 结果页（得分环形图 + 正确/错误统计 + 逐题解析列表，正确绿色错误红色高亮）。

#### #14 [Exam] 考试倒计时

实现倒计时组件。前端：ExamTimer.vue 组件（正常状态 / <5min 黄色警告 / <1min 红色脉冲动画 / 时间到触发自动交卷）。useExamTimer.js composable（reactive 响应式时间 / start/pause/stop 控制 / formatTime 格式化 MM:SS / timeUp 回调 / onUnmounted 自动清理定时器）。

#### #15 [Stats] 学习统计

实现学习数据可视化。后端：StatsController + StatsService。GET /overview 概览（totalQuestions / totalAnswered / overallAccuracy / studyDays / todayAnswered）。GET /accuracy/trend?days=7 正确率趋势 [{date, accuracy}]。GET /subject/progress 各科目进度。GET /daily/activity?days=30 每日活跃度 [{date, count}]。全部按 userId 过滤。通过 Mapper 自定义 SQL（GROUP BY + 日期函数）聚合查询。前端：stats.vue（概览卡片行 + ECharts 折线图 7 日趋势 + ECharts 柱状图 30 日活跃 + 科目进度条列表）。
