# 期末刷题助手 — 项目管理计划

## 一、项目概述

| 项目 | 说明 |
|------|------|
| **项目名称** | 期末刷题助手（Exam Review System） |
| **项目类型** | 前后端分离 Web 应用 |
| **技术栈** | Spring Boot 3 + Nuxt 3 + MySQL |
| **团队规模** | 4 人 |
| **开发周期** | 5 周 |
| **GitHub** | https://github.com/Lzz-04/exam-review-system |

### 项目目标

开发一个面向大学生的期末刷题平台，支持题库管理、章节练习、智能组卷、模拟考试、错题本、AI 辅助出题等完整学习闭环。

## 七、技术规范

### 后端规范

- 所有接口统一返回 `ApiResponse<T>` 格式：`{"code":200,"data":...,"message":"success"}`
- 异常统一由 `GlobalExceptionHandler` 处理，返回相应 HTTP 状态码
- 业务异常使用 `BusinessException`，自动映射为 400
- 数据库操作使用 MyBatis-Plus `LambdaQueryWrapper`，避免字符串拼接
- 写操作必须加 `@Transactional`

### 前端规范

- 所有 API 调用使用 `useApi()` 封装，不直接用 `fetch` / `$fetch`
- Token 由 `useApi.js` 自动注入，页面无需手动处理
- 页面级状态使用 Pinia Store，组件级状态使用 `ref` / `reactive`
- 通用 UI 组件放在 `components/`，页面放在 `pages/`

### Git 规范

- 分支命名：`feat/模块名`、`fix/问题描述`
- Commit 格式：`模块前缀: 简短描述`，如 `[Auth] 实现JWT登录`
- 每天至少 push 一次，避免代码丢失

| 接口 | 依赖方 | 说明 |
|------|--------|------|

### B 提供（C/D 依赖）

| 接口 | 依赖方 | 说明 |
|------|--------|------|

## 九、验收标准

### 核心流程验收

```
注册 → 登录 → 创建科目 → 创建章节 → 录入题目
    → 章节练习（顺序/随机/错题三种模式均可）
    → 组卷 → 考试（开始→暂停→恢复→交卷→看结果）
    → 错题自动入错题本 → 错题本标记掌握/移除
```

### 功能验收清单

| 模块 | 验收项 | 状态 |
|------|--------|------|
| 认证 | 注册 + 登录 + Token 过期跳转 | ☐ |
| 科目 | CRUD + 同名检测 + 有章节时禁止删除 | ☐ |
| 章节 | CRUD + 自动排序 + 有题目时禁止删除 | ☐ |
| 题目 | CRUD + 筛选 + 批量操作 + 随机抽题 + 级联删除 | ☐ |
| 导入 | TXT/DOCX/PDF/XLSX 四种格式导入 + 预览 | ☐ |
| 练习 | 顺序/随机/错题三种模式 + 即时反馈 + 错题同步 | ☐ |
| 考试 | 组卷 + 开始/暂停/恢复/交卷 + 评分 + 结果展示 | ☐ |
| 错题本 | 列表/筛选/统计 + 复习/掌握/移除 + 所有权校验 | ☐ |
| 统计 | 概览 + 正确率趋势 + 活跃度 + 科目进度 | ☐ |
| AI | DeepSeek 出题 + 预览 + 入库 | ☐ |
| 后台 | 用户列表 + 级联删除 | ☐ |
